Design

After conducting what I feel to be a sufficient amount of research, and testing a number of colour schemes on the website, I decided that a dark grey menu and footer, accompanied with a sky blue header, and white content area was the most fitting option for the website. I feel that the white text on the menu and footer allows for easy reading, and opted for the "black text, white background" approach for the main content area for accessibility reasons, as people with visual impairments may find other colour schemes difficult to read.

Inspiration for the navigation bar was taken from the W3 Schools website. However, I didn't feel that the tutorial itself provided me with a suitable navigation menu, particularly for mobile devices. Therefore, I coded the navigation bar such that the dark grey colour is replaced with the colour of the header on mobile devices. Also, the menu button follows the user as they scroll down the page - I felt that this design choice was appropriate, given that some it would prove particularly arduous to scroll to the very top to access the menu on some pages.

Every page on the website employs a responsive design. As mentioned before, the navigation bar's design changes depending on the device or screen resolution on which the website is being viewed. The header also changes - as the screen resolution decreases, the logo and the phone on the right hand side move closer together, eventually disappearing on the mobile layout, and placing focus on the logo by centring it.

Where pages employ a two-column layout, the second column is placed below the first column when viewing pages on devices as small as tablets - this is evident on the homepage, the Latest Releases page, and the Order Form page. On the Phones page, a different responsive layout is employed. As the screen resolution is reduced, fewer products are displayed on the same row

On the Latest Releases page, a responsive slideshow is used - for this, inspiration was taken from a tutorial on the W3Schools website. On devices with larger screen resolutions, the user is shown a picture of the featured device on the left hand side of the page, and a description on the right hand side - also on the right hand side are two buttons allowing the user to navigate through the slideshow. When the slideshow is shown on devices with resolutions of 1024x768 or lower, the content on the right hand side of the page is placed below the picture, and the picture becomes a central part of the page.

JavaScript

JavaScript is used widely throughout the site - on the Latest Releases page, the Phones page, and on the Order Form page.

On the Phones page, JavaScript is used to load a list of products from an XML file. After loading the XML file, the JavaScript code begins to populate a variable with HTML code, containing details of the phone, including the name, price, an image of the phone, and any features. The code looks for specific keywords in the features, and if it finds these then it places an image inside the div for the product.

On the Latest Releases page, the slideshow is powered by JavaScript. On this page, it is used to load the initial image, and to allow the user to navigate through the slideshow. The code also allows for the reduction of errors - if the user tries to go to the previous image when they are viewing the first image, then the code will load the final image; if the user tries to press the Next button on the final image, then they will be directed to the first image.

The Order Form contains the largest amount of JavaScript on the whole site. When the page loads, the first dropdown menu on the page is populated with a list of phones from the XML file. A small piece of JavaScript code also gets details of the current year, and produces a list of years for the expiry date dropdown menu.

The second piece of JavaScript code apparent on this page is triggered by a selection on the phone dropdown menu - when the user selects a phone, a div is populated with the phone's name and a picture, along with the upfront cost and the price per month; this piece of code also calculates the first payment, by adding together the two cost elements of the device. Also, when the user has selected the phone they wish to purchase, JavaScript code is employed to display the fields requesting their payment information.

The remaining JavaScript code on the page is used for the purpose of validation. If it is found that the user has not filled out a field, either at all or incorrectly, a message will be displayed below the field alerting the user of their error. Much of the code takes advantage of regular expression (Regex) statements to determine whether the input is valid - these statements are used on the phone number, e-mail address, postcode, and card number fields. The card number field is also able to determine the type of card the user is paying with, by extracting the first or second digits, depending on the card type.

The final piece of JavaScript code used on this page is run at the point of submission - if it picks up on any errors the user has made, then it refuses to submit the form, and points out all errors in the same way as the other validation techniques employed on the page.
